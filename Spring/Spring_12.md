# 세션
특별한 정보를 **서버에 **저장

HTTP <- 정보를 유지하는 기능X

## HTTP 통신 특징
1. connectionless(비연결지향)
>>- 더이상 연결을 지속적으로 하지 않음
2. Stateless(상태정보유지 안함)
>>- 요청을 응답하고 접속을 바로 끊어서 정보를 저장하지 않는다.
- ex)카톡보내고 카톡방 나감

이러한 HTTP 이슈 때문에 쿠키와 세션을 이용해서 간단한 정보를 임시로 서버에 저장한다.

# 쿠키
쿠키는 웹브라우저(크롬,사파리,웨일...)에 저장
- 프론트(javasscript)에서 저장
>>- 보안 낮음(세션이랑 비교했을때)

세션은 웹서버에 저장
- 세션은 백엔드(java)에 저장
>>- 보안 높음
